Main building blocks

  1. Role names stand for binary relations on elements and can be viewed as binary predicates.
    Example 1: Professor teaches Course; teaches here is a role name that relates Professors to Courses.


  2. Concept names represent sets of elements and can be viewed as unary predicates.

    3. Example 2: Course is a concept name that is interpreted as a set of elements such as Cs1 and Tech1. Professor is another concept name that is interpreted as a set of elements such as Paul and Sara.


Note 1:
To distinguish between the building blocks, we always write role names with lowercase letters and concept names with the first letter in uppercase.


Where to use the building blocks?

The concepts of a domain are described by concept descriptions, which are expressions that are built from concept names and role names using constructors?(a set of mathematical symbols e.g. ⊓, ∀, ∃, etc.) provided by the specific DL variant.

Example 3: to define the concept of “ Postgraduate students ”, we can use the following concept description:

Student ⊓ Postgraduate.

Example 4: to define the concept of “ students who like math ”, we can use the following concept description:

Student ⊓ ∃likes.Math.

In the previous examples Student, Postgraduate and Math are concept names, likes is a role name and ⊓ and ∃ are constructors.

Note 2:
When we say concept, concept names, and concept descriptions we mean three different things,
• "concept" means any class of things in the domain we are describing e.g. the concept of male doctors.
• "Concept name" means a term or an atomic concept e.g. Doctor.
• "Concept description" means expression that describes a concept using a concept name (an atomic concept) with a constructor e.g. ¬Doctor. or a compound concept that is built up from concept names an possibly role names with constructors e.g. Doctor ⊓ Male or ∃likes.Doctor.
